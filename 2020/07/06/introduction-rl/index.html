「RL基础-1」强化学习引言
发表于 2020-07-06| 更新于 2020-07-10|Reinforcement Learning
阅读量:

本文将默认已经了解马尔可夫奖励过程和马尔可夫决策过程的常见概念。

监督学习和强化学习

强化学习和监督学习都属于机器学习的一部分。

当下机器学习领域CV、NLP无疑是最热门并且研究发展最迅速的，而这两大领域上的Task往往都是使用大量的数据进行模型学习从而完成回归或者分类，也就是监督学习。

因此从监督学习出发，分析和强化学习之间的差异和联系。

SL和RL在训练数据上的区别

  • 监督学习：
    1. 不考虑强或弱监督，训练数据需要带标签，且数据之间需要满足独立同分布(i.i.d)
    2. 标签类别的语义区别要尽可能大，否则难以学习
  • 强化学习：
    1. 数据不具备i.i.d，因为训练数据往往具有时序相关性。
    2. 不能给出立即（大多时候）的反馈，要等到游戏/MDP过程结束才有反馈（作为标签）。

强化学习的特点

  • 试错探索(Trial-and-error exploration)，需要智能体进行大量的试错尝试。
  • 非即时反馈(Delayed reward)，一般不能在每个状态获得有效的反馈。
  • 数据间具有时序的关联，因为如果每组(instance)数据是基于在一个状态，那当前状态自然会依赖于之前的状态。
  • 智能体行为会影响后续的观测（类似于改变环境，这点也同时解释了上一点的时序相关性），因此在训练初期使得智能体能稳定生成有效数据，而不crash，是一个常见的问题。

强化学习的重要元素

  • 策略：智能体行为函数，用来选取下一步的动作。
  • 值函数：判断走入某状态对智能体的优劣（状态值函数），或是判断在一状态执行某动作对智能体的优劣（动作状态值函数，也称为Q函数）。
    • MDP中的状态值函数：vπ(s)≐Eπ[Gt∣St=s]=Eπ[∑k=0∞γkRt+k+1∣St=s], for all s∈Sv_{\pi}(s) \doteq \mathbb{E}_{\pi}\left[G_{t} \mid S_{t}=s\right]=\mathbb{E}_{\pi}\left[\sum_{k=0}^{\infty} \gamma^{k} R_{t+k+1} \mid S_{t}=s\right], \text { for all } s \in \mathcal{S}vπ​(s)≐Eπ​[Gt​∣St​=s]=Eπ​[∑k=0∞​γkRt+k+1​∣St​=s], for all s∈S
    • MDP中的动作状态值函数：qπ(s,a)≐Eπ[Gt∣St=s,At=a]=Eπ[∑k=0∞γkRt+k+1∣St=s,At=a]q_{\pi}(s, a) \doteq \mathbb{E}_{\pi}\left[G_{t} \mid S_{t}=s, A_{t}=a\right]=\mathbb{E}_{\pi}\left[\sum_{k=0}^{\infty} \gamma^{k} R_{t+k+1} \mid S_{t}=s, A_{t}=a\right]qπ​(s,a)≐Eπ​[Gt​∣St​=s,At​=a]=Eπ​[∑k=0∞​γkRt+k+1​∣St​=s,At​=a]
  • 模型：智能体对所处环境世界的理解，一般用于在给定状态、动作下预测环境反馈（包括状态转移概率和奖励值）。

强化学习的一些分类

​ 根据强化学习求解类型，可以进行一些分类，比如Policy-based、Value-based；Model-based、Model-free，首先看Policy-based、Value-based。

Policy-based vs Value-based

​ Policy-based的RL方法，也就是计算策略向量，从策略向量中采样动作进行control。而Value-based的RL方法就是计算值函数，再贪心地选取邻接状态中值函数最高的动作进行control。下面的图直观地展示了这两种方法的运作逻辑。

​ 可以看出，Value-based的方法会学习每个状态的值函数，而策略不会学习值函数，只会显式地学习策略。至于一个经典的RL方法Actor-Critic即会学习值函数又会学习策略，这类方法就即属于Policy-based也属于Value-based。

  • Value-based RL
    • 显式学习值函数
    • 通过值函数隐式地得到策略
  • Policy-based RL
    • 显式学习策略，但不学习值函数

Model-based和Model-free

​ 如果没有学习完整的强化学习基础知识，这点还是比较难以理解。

​ 因此本文仅先从两者的不同应用场景去解释。

​ 强化学习目前的应用领域有游戏、自动驾驶、机器人等等。相对游戏这类环境本身就建立在仿真系统下的任务，自动驾驶、机器人这类任务是需要最终部署在现实中的。我们知道RL需要靠大量的试错采样来学习，然而自动驾驶和机器人这种任务，广泛(extensive)试错的成本太高昂，稍不注意可能会导致硬件损坏甚至威胁研究者的人身安全。

​ 此外，想像游戏一样，将机器人这类任务环境建模在仿真系统下，需要庞大的工程量，或者稍微复杂的任务以目前的技术都是不可能实现的。

​ 也就是说，自动驾驶、机器人这类的大部分任务不管是从现实采样，还是建立仿真环境采样，都是很难（成本太高）做到的。

​ 因此，Model-based的方法就是考虑用函数近似的方法，建立一个函数近似的模型(Model)，去学习、认识环境，使得给定当前状态、动作时，能输出环境的反馈，这样就既避免了从现实中采样，又避免了构建工程量巨大的仿真环境。至于Model-free，就容易理解了，也就是不构建函数近似Model，需要环境的反馈就直接和环境进行交互来得到。

​ 从应用场景这个角度粗浅地认识，Model-based就更多针对自动驾驶、机器人这类任务，而Model-free就更多针对游戏这类任务。更多理论上的区别会在后面一篇专门解读RL中易混淆的概念中写出。

MDP迭代求解值函数

首先来看看相对于MRP，MDP有哪些特性。

MRP与MDP

​ 强化学习的基础模型是MDP马尔可夫决策过程<S,A,P,R,γ><\mathcal{S}, \mathcal{A}, \mathcal{P}, \mathcal{R}, \gamma><S,A,P,R,γ>，而马尔可夫奖励过程MRP<S,P,R,γ><\mathcal{S}, \mathcal{P}, \mathcal{R}, \gamma><S,P,R,γ>，是不考虑策略π\piπ的MDP。因此MRP上只有值函数的计算。

​ 此外，仅需输入状态就可以得到MRP上的环境反馈P(s′∣s)P\left(s^{\prime} \mid s\right)P(s′∣s)和R(s)R(s)R(s)，而MDP的输入还考虑了动作才能得到P(s′∣s,a)P\left(s^{\prime} \mid s, a\right)P(s′∣s,a)和R(s,a)R(s, a)R(s,a)，后者可以通过以π(a∣s)\pi(a|s)π(a∣s)进行weighted sum转换为前者。即：

P(s′∣s)=∑a∈Aπ(a∣s)P(s′∣s,a)R(s)=∑a∈Aπ(a∣s)R(s,a)\begin{aligned} P\left(s^{\prime} \mid s\right) &=\sum_{a \in A} \pi(a \mid s) P\left(s^{\prime} \mid s, a\right) \\ R(s) &=\sum_{a \in A} \pi(a \mid s) R(s, a) \end{aligned} P(s′∣s)R(s)​=a∈A∑​π(a∣s)P(s′∣s,a)=a∈A∑​π(a∣s)R(s,a)​

​ 这样的一种将动作aaa给marginalization掉的做法，在之后的RL理论推导中会经常用到。

例子1：状态值函数和动作状态值函数

vπ(s)=∑a∈Aπ(a∣s)qπ(s,a)(2.1)v^{\pi}(s)=\sum_{a \in A} \pi(a \mid s) q^{\pi}(s, a) \tag{2.1} vπ(s)=a∈A∑​π(a∣s)qπ(s,a)(2.1)

例子2：状态值函数和动作状态值函数的贝尔曼方程

​ 状态值函数的Bellman expectation equation

vπ(s)=Eπ[Gt∣st=s]=Eπ[Rt+1+γvπ(st+1)∣st=s](2.2)\begin{aligned} v^{\pi}(s)&=\mathbb{E}_{\pi}\left[G_{t} \mid s_{t}=s\right] \\ & =\mathbb{E}_{\pi}\left[R_{t+1}+\gamma v^{\pi}\left(s_{t+1}\right) \mid s_{t}=s\right] \end{aligned} \tag{2.2} vπ(s)​=Eπ​[Gt​∣st​=s]=Eπ​[Rt+1​+γvπ(st+1​)∣st​=s]​(2.2)

​ 动作状态值函数的贝尔曼方程

qπ(s,a)=Eπ[Gt∣st=s,At=a]=Eπ[Rt+1+γqπ(st+1,At+1)∣st=s,At=a](2.3)\begin{aligned} q^{\pi}(s, a)&=\mathbb{E}_{\pi}\left[G_{t} \mid s_{t}=s, A_{t}=a\right] \\ & =\mathbb{E}_{\pi}\left[R_{t+1}+\gamma q^{\pi}\left(s_{t+1}, A_{t+1}\right) \mid s_{t}=s, A_{t}=a\right] \end{aligned} \tag{2.3} qπ(s,a)​=Eπ​[Gt​∣st​=s,At​=a]=Eπ​[Rt+1​+γqπ(st+1​,At+1​)∣st​=s,At​=a]​(2.3)

qπ(s,a)=Rsa+γ∑s′∈SP(s′∣s,a)vπ(s′)q^{\pi}(s, a)=R_{s}^{a}+\gamma \sum_{s^{\prime} \in S} P\left(s^{\prime} \mid s, a\right) v^{\pi}\left(s^{\prime}\right) qπ(s,a)=Rsa​+γs′∈S∑​P(s′∣s,a)vπ(s′)

预测与控制

​ Value-based的MDP上一般有两个任务，先估计值函数，再计算最优策略。前者称为预测(prediction)，后者称为控制(control)。

Prediction任务

定义：输入MDP和策略π\piπ，输出值函数vπv^\pivπ。预测的意义等同于评估一个策略π\piπ的价值，因此也prediction任务也称为策略估计(Policy evaluation)。

一般性过程：利用贝尔曼方程迭代求解状态值函数，迭代方法有蒙特卡洛、时间差分、动态规划等。

vt+1(s)=∑a∈Aπ(a∣s)(R(s,a)+γ∑s′∈SP(s′∣s,a)vt(s′))v_{t+1}(s)=\sum_{a \in \mathcal{A}} \pi(a \mid s)\left(R(s, a)+\gamma \sum_{s^{\prime} \in \mathcal{S}} P\left(s^{\prime} \mid s, a\right) v_{t}\left(s^{\prime}\right)\right) vt+1​(s)=a∈A∑​π(a∣s)(R(s,a)+γs′∈S∑​P(s′∣s,a)vt​(s′))

其中ttt表示迭代的次数。在每一次迭代，要对每一个状态sss都进行更新（上面公式只展示了一个状态）。

Control任务

定义：输入MDP，输出最优值函数v∗v^*v∗和最优策略π∗\pi^*π∗

一般性过程：

  1. 由状态值函数计算动作状态值函数(state-action value)：

    qπi(s,a)=R(s,a)+γ∑s′∈SP(s′∣s,a)vπi(s′)q^{\pi_{i}}(s, a)=R(s, a)+\gamma \sum_{s^{\prime} \in S} P\left(s^{\prime} \mid s, a\right) v^{\pi_{i}}\left(s^{\prime}\right) qπi​(s,a)=R(s,a)+γs′∈S∑​P(s′∣s,a)vπi​(s′)

  2. (贪心地)得到动作：

π∗(a∣s)={1, if a=arg⁡max⁡a∈Aq∗(s,a)0, otherwise \pi^{*}(a \mid s)=\left\{\begin{array}{ll} 1, & \text { if } a=\arg \max _{a \in A} q^{*}(s, a) \\ 0, & \text { otherwise } \end{array}\right. π∗(a∣s)={1,0,​ if a=argmaxa∈A​q∗(s,a) otherwise ​

​ 然而需要考虑一个问题：以上的一般性过程需要对每个状态和动作进行枚举并计算q函数，才能得到策略，复杂度还是很高的。而采用使用策略搜索(Policy search)可以更有效率，例如Policy iteration、Value iteration。

​ 因此control任务也可以称为策略提升(Policy improvement)。

​ 这一步迭代收敛的判定条件是Bellman optimality equation，vπ(s)=max⁡a∈Aqπ(s,a)v^{\pi}(s)=\max _{a \in \mathcal{A}} q^{\pi}(s, a)vπ(s)=maxa∈A​qπ(s,a)。
